突破原点盘旋

两人间的唯一区别，你有更多选择，而我只有一个选择。

从更多选择出发，首先踌躇，犹豫不决，迟迟不能转换成为行动。

从唯一选择出发。看到底是否可行，怎么执行，然后开始第一步，做下去直到彼岸。

每一个优秀的人，都不是与生俱来带着光环的，也不一定是比别人幸运。他们只是在任何一件小事上，都对自己有所要求，不因舒适而散漫放纵，不因辛苦而放弃追求。雕塑自己的过程，必定伴随着疼痛与辛苦，可那一锤一凿的自我敲打，终究能让我们收获一个更好的自己。

见过太多原点盘旋，在原点耗费太多时间精力确认最佳方案。什么是最佳，可能需要在实践中逐渐验证。

在原点盘旋阶段，也有效率要素。有效率地确认当下的唯一选择，不能确认的时候抛硬币选择其一，开始行动，胜过原地打转转琢磨哪一个选择最棒。

清单革命061-寻找生命中的放大器

日常工作和生活中，我们经常遇到那一些当初我们认为没有那么起眼的人，后来奋斗的远超我们，我们很不服气。但是除了吐槽之外，又能怎么样呢？除了在背后八卦一点往事之外，我们又可以做什么呢？

现在我们一起来探讨这个现象，看看有没有可以借鉴的经验。

我这些年非常痛的一个领悟，就是你在上一个接点，稍微努力一下，爬高一点点。在下一个节点会被再次放大，然后再下一个节点再次放大。

回头再看最初的那个节点稍微爬高的那一点点，根本无足轻重，但是效能后来被无限放大。

我们把这个过程可以分解成为三步

第一步，努力在当下，现在多努力一点点，赢得现在的一个小台阶

第二步，放大，进入到下一个阶段，进一步放大

第三步，新台阶，继续新的努力在当下

上述三步中有两个要点。

第一个要点增量思维，是上一个阶段的努力，在下一个阶段增量。至少是在同一个方向上进行向上。也就是说，存在着水平方向的叠加式的努力，也存在着垂直方向的上升。

第二个要点当下思维，不管未来如何，我依旧要把今天的事情做好，不管未来走向何方，不管我现在的努力在未来具有多大的价值，我今天就是要去做。

总结

识别出来上述三个步骤，两个要点，希望有助于我们过好现在的生活，让现在的努力在未来持续放大。

build with taro, vant-weapp

关键词 Taro vant-weapp

在开发小程序中，尝试使用Taro作为基础框架，引入vant-weapp，Taro-UI。 引入后者相对来说要简单一些， 但是引入前者比较麻烦，我引入成功后分享出来， 希望帮助到更多的朋友，降低整合成本。

  1. Taro ^1.1.0-beta.2 或 ^1.1.0
  2. Type Script

整合有赞的组件库 vant-weapp。 可以做整合的示例。

Taro 官网地址: https://github.com/NervJS/taro

Vant-weapp文档地址: https://youzan.github.io/vant-weapp

安装Taro的新版本

  1. 使用taro init myApp
  2. 修改package.json中的taro相关包的版本号*^1.1.0-beta.2
  3. 运行npm install

整合要点

  1. 下载vant-weapp/dist保存在src/components/vant-weapp/dist
  2. 在首页index.jsx中修改usingComponents,声明引入的组件
  3. 在render方法中直接组件比如van-button
