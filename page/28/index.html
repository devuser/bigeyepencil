思维模型

喜洲严家

造访喜洲商帮领袖严家花园

喜洲隶属大理，处在茶马古道和某某官道交叉口，元朝攻降大理城（时称羊苴宜城）后易地修筑喜洲城，驻扎于此。清末民初，诞生富可敌国的严氏家族。曾经有崔严尹杨四大家，还有八中家，十二小家。尤以严家为最，抗日期间曾经捐献三架飞机，去世后得到常公挽联。

游历喜洲严家老宅子，分析短时间内富可敌国的关键要素，总结如下四条。

  1. 拥有先进文化知识的领袖，有文化的传统士绅进入缺乏知识的马帮商人阶层 ，降维打击

  2. 技术革命，率先引入机械制作沱茶，互联网技术

  3. 商业模式，跨省的产供销，供应链

  4. 靠谱的产品，沱茶品质稳定，容易保证质量，支撑远距离运输，易储存，产品思维

go run 使用

  • go run [fileName].go 用于执行go的源代码 如go run helloworld.go
  • go run [fileName].go -argK=argV 带参数[argK]执行go的源代码

笔者常用的命令方式

go run 常用标记

  • -a：强制编译相关代码，无论是否编译过
  • -v：列出被编译的代码包的名称
  • -a -v：列出所有被编译的代码包的名称
  • -p n：并行编译，其中n为并行的数量 如-p 2
  • -n：打印编译过程中所需运行的命令，但并不执行这些命令
  • -x：打印编译过程中所需运行的命令，并执行这些命令
  • -work：显示编译时创建的临时工作目录的路径，并且不删除这个临时工作目录

使用方式：如 go run -v [fileName].go

常用第三方包

常用第三方包

包的介绍

包类似Java中概念，jar 是源代码管理，分发的最小单位。

目前多数包来自 Github 官方包来自 golang.org/x/...

可以在如下网址查询到高频使用的第三方包清单 https://godoc.org

常用清单

  1. github.com/astaxie/beego 成熟稳定的Web框架，包含更过的Web Framework特性 国内开发者 astaxie
  2. github.com/go-redis/redis 连接Redis
  3. github.com/gomodule/redigo/redis 连接Redis
  4. github.com/jinzhu/gorm 数据库ORM框架，类似Java领域的Hibernate或MyBatis 国内开发者
  5. github.com/sirupsen/logrus 日志框架，类似Java领域的log4j
  6. github.com/robfig/cron 定时任务，类似Java领域的Quartz
  7. github.com/gin-gonic/gin 精巧高效的Web框架，多用于提供Web服务

编译过程一般来说分成如下几步

  1. 前从 git/svn 服务器抓取最新的版本

  2. 清理现场

  3. 获得时间戳，以便在后续编译过程中带入作为版本号的后缀

  4. 编译，携带若干参数（资源文件、不含调试信息、版本号，目标文件名称）

  5. 复制到特定文件夹

  6. 连同配置文件一起压缩，压缩文件带有上述时间戳

  7. 导出源代码，并压缩，压缩文件带有上述时间戳

  8. 清理现场

上述步骤连续做下来，没有一个工程师可以可以连续无差错做两遍。 这样的场景适合创建一个批处理文件，如果在Linux下的话就是Shell脚本文件。
